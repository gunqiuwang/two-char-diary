## 一、项目概述

**产品定位**：每天用「两个字」和「一个爻卦」记录当天。

**核心理念**：二字是主体，卦象是注脚。

**目标用户**：喜欢极简记录的人。

**设计风格**：Kami — 暖米白底、墨蓝点缀、衬线体、克制留白。

## 四、已知问题（需修复）

### 4.1 高优先级

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 分享页二字偶发丢失 | 🔴 高 | `goToShare()` 已改为从存储读取 `d.words`，但可能仍有边界情况。建议统一从存储读取所有数据 |
| Canvas 海报二字可能为空 | 🔴 高 | `generateCard()` 中 `this.data.words` 可能为空，需要从存储兜底 |
| 爻辞数据不完整 | 🟡 中 | `HEX_DATA` 只有卦名+卦辞，没有完整六爻爻辞。之前有个带爻辞的版本被覆盖了 |

| 问题 | 严重度 | 说明 |
|------|--------|------|

| 米字框输入后没有清除输入框残留 | 🟡 中 | 输入完二字后 input 值可能残留 |
| 年历页点击无记录日期无反馈 | 🟢 低 | 可加个 toast 提示 |

### 4.3 低优先级

## 五、设计规范

### 色板

| 变量 | 色值 | 用途 |
|------|------|------|
| 背景 | `#f5f4ed` | 暖米白，页面底色 |
| 主文字 | `#2F312B` | 深墨色，二字主体 |
| 次文字 | `#4A4A45` | 暖灰，爻线、卦名 |
| 辅助文字 | `#8A8A82` | 橄榄灰，原文 |
| 元数据 | `#B8B6AE` | 石灰色，日期、标签 |
| 边框 | `#E8E5DC` | 暖米色，分割线 |
| 铜钱 | `#C4B08C → #A89870` | 暖铜渐变 |
| 强调 | `#B8A88A` | 铜色，动爻点、分享入口 |

### 字体

| 场景 | 字体 |
|------|------|
| 二字主体 | `STSong`, `Songti SC`, `PingFang SC`, serif |
| 卦名 | `STSong`, `Songti SC`, serif |
| 正文 | 系统默认 |
| 辅助文字 | 系统默认 |

### 间距

- 页面内边距：`60rpx`
- 二字字距：`12rpx - 16rpx`
- 爻线间距：`8rpx - 10rpx`
- 底部导航高度：`120rpx`

## 七、页面状态机

### 首页 castingState

```
idle → casting → result
  ↑                |
  └────────────────┘  (点击铜钱重新起课)
```

### 铜钱状态

| 状态 | 显示 | 透明度 | 可点击 |
|------|------|--------|--------|
| idle | 三枚叠加 | 1.0 | ✅ |
| casting | 分散抛起动画 | 1.0 | ❌ |
| result | 三枚叠加 | 0.5 | ✅ |

## 十、交接注意事项

1. **不要重写整个项目**：现有功能基本可用，只需修复已知问题 + 补全爻辞数据
2. **二字永远是主体**：所有页面都要保持"二字 > 卦象 > 原文"的层级
3. **铜钱始终显示**：不要用 `wx:if` 隐藏铜钱，用 CSS opacity 控制
4. **轻量原则**：首页不要放爻辞列表，那是详情页的内容
5. **存储 key 统一用 `diary_data`**：所有页面都从这个 key 读写
6. **设计风格不要改**：Kami 色板和字体已定义好，不要引入新颜色
7. **旧 history 页面可删除**：已被 calendar 替代
